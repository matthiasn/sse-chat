Template Structure

You can contribute your own templates to Activator by simply creating a new project in the correct structure and then publishing it. This tutorial will walk you through the structure of a template and show you how to publish templates.

There are three main parts to a template:

  1. A Project with an SBT build
  2. A Tutorial for the Project
  3. Template Metadata

These three pieces are combined into a project that can be used in Activator and as a regular SBT project. With Activator, when a user creates a new app from a template, the tutorial and the metadata are not included in the new project.

Project Structure

The project for an Activator template should have the following layout:

  • /app contains the Java and Scala app source code
  • /project contains the SBT metadata and optionally any SBT plugin definitions
  • /build.sbt or /project/Build.scala contains the app's SBT build definition
  • /test contains the Java and Scala test source source
  • /conf optionally contains the /conf/routes and /conf/application.conf files for Play apps

The project must set a required SBT version in the /project/build.properties file, like: 

sbt.version=0.12.3

To create consistency across all templates in Activator, the default /build.sbt follows the Play defaults and changes the main source directories from /src/main/java and /src/main/scala to just /app. It also changes the main test directories from /src/test/java and /src/test/scala to just /test. Please keep these settings in place since users can easily change them after creating a new project.

Tutorial Structure

The tutorial you are reading right now comes from the /tutorial/index.html file.

Each section of the tutorial must be in a <div> tag that is inside the <body> tag. Each section must have a single <h2> tag that contains the section's title.

The tutorial can provide shortcut links to places in Activator by using an anchor link (#) and the shortcut class, like: 

<a href="#code/tutorial/index.html" class="shortcut">/tutorial/index.html</a>

Code blocks can use <pre><code></code></pre> tags, like: 

<pre><code>println("hello, Scala!")
val foo = "Bar"</code></pre>

In-line code can use just the <code></code> tag, like: 

The <code>val foo = "Bar"</code> statement sets <code>foo</code> to <code>Bar</code>

Metadata Structure

Activator templates must provide an activator.properties file which contains the template metadata. The required properties are:

  • name - A globally unique, URL friendly name for the template
  • title - A human readable name for the template
  • description - A plain text description for the template
  • tags - A comma-seperated list of tags for the template
These values will be verified when you publish your template.

Publishing

At this time templates must first be published publicly on GitHub. Then they can be added to Activator through the Activator Contribution page. There you can enter a Git URL, like: 

git://github.com/typesafehub/activator-template-template.git
This will use the master branch of your repo. Soon publishing will happen automatically whenever your GitHub repo changes (for the specified branch). Until then you will need to go through this process every time you want to publish a new version.
